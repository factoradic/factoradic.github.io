Books list is created using ng-repeat directive.

List of authors is coded by hand to show how to create links without {{ index }} iterator and ng-repeat directive.

BookController use a extra property urlName. It saves rendering time when we display link and when we go to linked page. Difference in loading time is caused by lack of string normalization process, but we have to specify one additional property for our data objects. This method is widely used in many content management systems. A big plus is that we have a direct impact on the url, for example in news sites, we usually don't want to embed whole article title in page address.

AuthorController use a string normalization process to determine which info has to be rendered.

Please remember that links must be unique. In the case of very large applications, where data are modified by many people it's hard to control every link and object. User experience is your main concern. You might solve this problem by getting all matched objects and rendering them. I presented this method in BookController.
